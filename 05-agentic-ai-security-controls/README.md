# Agentic AI Security Controls

## Project Summary

This project demonstrates security controls that should be considered when AI agents interact with IAM, GRC, ticketing, audit, or security workflows.

## Business Problem

AI agents can increase productivity, but they can also introduce risks if they have excessive permissions, weak approval controls, poor logging, or access to sensitive systems without oversight.

## Agent Risk Areas

- Excessive tool permissions
- Prompt injection
- Unauthorized data access
- Poor audit logging
- Weak human approval gates
- Inaccurate or unsupported recommendations
- Over-automation of security decisions

## Skills Demonstrated

- AI governance
- Agentic AI security
- Threat modeling
- Human-in-the-loop control design
- Logging and monitoring requirements
- Security test scenario design
